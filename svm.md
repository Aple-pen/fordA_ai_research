## SVM 정리: 개념, 파라미터, 그리고 이번 실험 결과 해석

### 1. SVM이란 무엇인가?

SVM(Support Vector Machine)은 **클래스를 가장 잘 구분하는 결정 경계(decision boundary)** 를 찾는 지도학습 분류 모델입니다. 핵심 아이디어는 단순히 두 클래스를 나누는 선 또는 면을 찾는 것이 아니라, **두 클래스 사이의 간격(margin)을 최대화하는 경계** 를 찾는 것입니다.

즉, SVM은 다음 질문에 답하려는 모델입니다.

- 정상과 비정상을 나누는 경계는 어디에 두는 것이 가장 안정적인가?
- 새로운 데이터가 들어왔을 때, 어느 쪽 클래스에 더 가깝다고 판단할 수 있는가?

SVM은 특히 다음 상황에서 자주 강력한 성능을 보입니다.

- 데이터 차원이 비교적 높을 때
- 클래스 경계가 단순 직선이 아닐 때
- 표본 수가 아주 크지 않지만 경계가 비교적 뚜렷할 때

---

### 2. Margin과 Support Vector

SVM에서 가장 중요한 개념은 **margin** 과 **support vector** 입니다.

- **Margin**: 두 클래스 사이의 여유 폭
- **Support Vector**: 결정 경계를 실제로 밀고 있는 핵심 샘플

경계를 결정하는 것은 모든 데이터가 아니라, 경계 가까이에 있는 일부 샘플들입니다. 이 샘플들이 바로 support vector입니다. 그래서 모델 이름도 Support Vector Machine입니다.

Margin이 넓을수록 모델은 작은 잡음이나 새로운 데이터 변화에도 더 안정적으로 동작할 가능성이 큽니다.

---

### 3. 선형 SVM과 비선형 SVM

#### 3.1 선형 SVM

선형 SVM은 데이터가 직선(2D), 평면(3D), 또는 초평면(high-dimensional)으로 잘 나뉜다고 가정합니다.

예를 들어 다음처럼 한 줄로 나눌 수 있으면 선형 SVM이 잘 작동합니다.

- 왼쪽 영역: 비정상
- 오른쪽 영역: 정상

하지만 실제 데이터는 선형으로 깔끔하게 분리되지 않는 경우가 많습니다. FordA처럼 시계열 패턴이 복잡한 데이터는 특히 그렇습니다.

#### 3.2 비선형 SVM

비선형 SVM은 **커널(kernel)** 을 이용해서 원래 공간에서는 잘 안 나뉘는 데이터를, 더 높은 차원의 특징 공간에서 나누기 쉽게 만듭니다.

이번 실험에서 `linear`보다 `rbf`가 훨씬 성능이 좋았다는 것은, 현재 PCA 특징 공간에서도 **클래스 경계가 여전히 비선형적** 이라는 뜻으로 해석할 수 있습니다.

---

### 4. Kernel이란 무엇인가?

Kernel은 SVM이 데이터를 어떤 방식으로 분리할지 결정하는 함수입니다.

이번 노트북에서 비교한 커널은 다음과 같습니다.

- **linear**: 직선/평면 형태의 선형 경계
- **rbf**: 국소적인 비선형 경계를 만들 수 있는 가장 대표적인 커널
- **poly**: 다항식 형태의 경계
- **sigmoid**: 신경망 활성함수와 비슷한 형태의 경계

#### 이번 실험 결과 해석

- `linear`: 약 50%
- `poly`: 약 52%
- `sigmoid`: 약 52%
- `rbf`: 약 88%

이는 사실상 **RBF만 유의미하게 작동했다** 는 뜻입니다.

즉, 현재 FordA 데이터의 구조는 단순 선형 분리가 아니고, **복잡한 곡선형 경계** 가 필요하다는 의미입니다.

---

### 5. SVM의 주요 파라미터

#### 5.1 `kernel`

결정 경계의 형태를 정합니다.

- `kernel='linear'`: 선형 분리 시도
- `kernel='rbf'`: 비선형 분리 시도
- `kernel='poly'`: 다항식 형태 경계
- `kernel='sigmoid'`: sigmoid 형태 경계

이번 실험에서는 `rbf`가 가장 적절했습니다.

#### 5.2 `C`

`C`는 **오분류를 얼마나 강하게 벌줄지** 결정하는 파라미터입니다.

- `C`가 작으면:
  모델이 더 부드러운 경계를 허용합니다. 일부 오분류를 감수하면서 일반화를 중시합니다.
- `C`가 크면:
  훈련 데이터를 더 엄격히 맞추려고 합니다. 과적합 위험이 커질 수 있습니다.

직관적으로 보면 `C`는 **'실수 허용 정도'** 를 조절하는 값입니다.

#### 5.3 `gamma`

`gamma`는 RBF 커널에서 **한 샘플이 주변에 미치는 영향 범위** 를 조절합니다.

- `gamma`가 작으면:
  영향 범위가 넓어져서 경계가 부드럽고 완만해집니다.
- `gamma`가 크면:
  영향 범위가 좁아져서 경계가 매우 복잡해지고 국소 패턴에 민감해집니다.

즉, `gamma`는 **경계의 굴곡 정도** 를 결정합니다.

이번 결과에서 `gamma=0.1`은 거의 대부분 성능이 무너졌습니다. 이는 경계가 너무 복잡해져서 일반화가 깨졌다고 볼 수 있습니다.

#### 5.4 `random_state`

재현성을 위한 설정입니다. 실험을 다시 했을 때 같은 결과를 최대한 유지하도록 돕습니다.

### 6. 이번 노트북에서 PCA를 먼저 쓰는 이유

원본 FordA 데이터는 500차원입니다. 차원이 너무 높으면 다음 문제가 생길 수 있습니다.

- 계산량 증가
- 노이즈 차원 포함 가능성 증가
- SVM이 불필요한 방향까지 학습할 위험

그래서 먼저 PCA/SVD 기반 투영을 통해 주요 정보가 있는 축만 남긴 뒤 SVM을 적용했습니다.

현재 노트북의 흐름은 다음과 같습니다.

1. 원본 시계열 데이터 스케일링
2. SVD 수행
3. 주성분 축으로 학습/테스트 데이터 투영
4. 그 투영 결과에 SVM 적용

즉, 여기서 SVM은 **원본 500차원 전체를 직접 보는 것이 아니라, PCA/SVD로 압축된 특징 공간에서 분류** 하고 있습니다.

---

### 6.1 `Vt_scaled`는 무엇이며, 왜 테스트 데이터에 곱하는가?

이 부분은 PCA/SVD를 처음 적용할 때 가장 많이 헷갈리는 지점입니다.

현재 노트북에서는 학습 데이터 행렬을 다음과 같이 두고 있습니다.

$$
X_{train} \in \mathbb{R}^{n_{samples} \times n_{features}}
$$

즉,

- 행(row): 샘플
- 열(column): feature

이때 SVD를 적용하면

$$
X_{train} = U S V^T
$$

가 되고, 각 행렬의 의미는 다음과 같습니다.

- `U`: 각 학습 샘플이 저차원 축에서 어디에 놓이는지 나타내는 좌표
- `S`: 각 축의 중요도(특이값)
- `V^T` (`Vt_scaled`): 원래 feature 공간에서의 주성분 축 방향

즉, `Vt_scaled`는 단순히 어떤 수치 행렬이 아니라, **학습 데이터로부터 추출한 "주성분 축의 방향"** 을 담고 있습니다.

#### `Vt_scaled`를 열공간이라고 생각하면 왜 헷갈리는가?

수학적으로 SVD를 배울 때는 보통 다음처럼 정리합니다.

- `U`의 열벡터: $X$의 column space와 관련
- `V`의 열벡터: $X$의 row space와 관련

그런데 머신러닝에서는 행렬을 "샘플 x feature" 구조로 쓰기 때문에, `V`의 열벡터를 **feature 공간의 basis(기준축)** 로 해석하는 것이 더 직관적입니다.

따라서 현재 노트북에서는 `Vt_scaled`를 다음처럼 이해하는 것이 가장 자연스럽습니다.

- `Vt_scaled[:k, :]`: 원래 500차원 feature 공간 안에서 찾은 상위 $k$개 주성분 축

#### 학습 데이터는 왜 `U * S`로 표현되는가?

학습 데이터는 이미 SVD 분해를 했기 때문에, 상위 $k$개 성분만 쓰면 저차원 좌표가

$$
Z_{train}^{(k)} = U[:, :k] S[:k]
$$

로 바로 주어집니다.

코드에서는 이것을 다음과 같이 계산했습니다.

```python
X_train_pca_multi = U_scaled[:, :n_components_90] * S_scaled[:n_components_90]
```

이 값은 학습 샘플들이 주성분 공간에서 가지는 좌표입니다.

#### 테스트 데이터는 왜 `Vt_scaled`에 dot product 하는가?

테스트 데이터는 SVD를 새로 한 것이 아니므로 `U_test`가 없습니다. 하지만 학습 데이터에서 이미 찾아낸 주성분 축은 있습니다. 그 축이 바로 `Vt_scaled`입니다.

따라서 테스트 데이터도 **학습 데이터가 만든 동일한 좌표계로 들어와야** 합니다. 이를 위해 테스트 데이터를 그 축에 사영(projection)합니다.

수식으로 쓰면,

$$
X_{test}^{(k)} = X_{test} V_k
$$

여기서

$$
V_k = Vt\_scaled[:k, :]^T
$$

이므로 코드가 다음처럼 됩니다.

```python
X_test_pca_multi = np.dot(x_test_scaled, Vt_scaled[:n_components_90].T)
```

즉, 테스트 데이터를 **학습 데이터에서 얻은 주성분 축으로 좌표 변환** 하는 것입니다.

#### 왜 테스트 데이터에 대해 SVD를 다시 하면 안 되는가?

테스트 데이터에 대해 별도로 SVD를 수행하면 테스트 데이터만의 새로운 축이 만들어집니다. 그러면 다음 문제가 생깁니다.

- 학습 데이터가 있던 좌표계와 테스트 데이터 좌표계가 달라짐
- SVM은 학습 좌표계 기준으로 경계를 배웠는데, 테스트는 다른 좌표계로 들어오게 됨
- 결과적으로 학습/테스트 간 특징 공간이 일치하지 않게 됨

즉, 분류 모델은 **같은 좌표계에서 학습하고 같은 좌표계에서 평가** 해야 합니다.

#### 비유로 이해하기

이 과정을 지도 비유로 생각하면 더 쉽습니다.

- 학습 데이터로 "북쪽/동쪽" 축을 정한다 = `Vt_scaled`
- 학습 데이터는 그 축 기준으로 좌표가 이미 계산되어 있다 = `U * S`
- 테스트 데이터도 같은 축 기준으로 좌표를 다시 읽어야 한다 = `X_test @ V`

만약 테스트 데이터만 따로 축을 새로 정하면, 같은 위치도 서로 다른 기준으로 표현하게 되므로 직접 비교가 불가능합니다.

#### 한 줄 요약

- `U * S`: 학습 샘플의 주성분 좌표
- `Vt_scaled`: feature 공간에서의 주성분 축
- `X_test @ V`: 테스트 데이터를 그 동일한 축으로 투영한 결과

즉, **테스트 데이터를 학습 데이터와 같은 주성분 좌표계에 올려놓기 위해 `Vt_scaled`와 dot product를 하는 것** 입니다.

### 7. Classification Report는 어떻게 읽는가?

기본 RBF SVM 결과는 다음과 같았습니다.

- Test Accuracy: 약 87.80%
- 비정상(`-1`) precision/recall/f1: 약 0.88
- 정상(`1`) precision/recall/f1: 약 0.87

이 수치의 의미는 다음과 같습니다.

#### 7.1 Accuracy

전체 정답률입니다.

- 약 88%라는 것은
  테스트 샘플 1320개 중 약 88%를 맞췄다는 뜻입니다.

#### 7.2 Precision

모델이 어떤 클래스로 예측한 것 중 실제로 맞은 비율입니다.

예를 들어 비정상 precision이 0.88이면:
- 모델이 '비정상'이라고 한 샘플들 중 88%가 실제 비정상이라는 뜻입니다.

#### 7.3 Recall

실제 그 클래스인 샘플 중에서 모델이 찾아낸 비율입니다.

예를 들어 비정상 recall이 0.88이면:
- 실제 비정상 중 88%를 놓치지 않고 찾아냈다는 뜻입니다.

#### 7.4 F1-score

Precision과 Recall의 균형을 함께 보는 지표입니다.

클래스 불균형이 있거나, 단순 accuracy만으로는 부족할 때 특히 중요합니다.

- `rbf`만 높은 성능을 보여, **현재 데이터는 비선형 경계가 핵심** 이라는 점을 시사함

따라서 이후 튜닝도 `rbf`에 집중하는 것이 타당합니다.

### 10. `n_components + SVM` 동시 튜닝 결과 해석

이번 확장 실험에서는 다음을 동시에 바꿨습니다.

- PCA 차원 수 `n_components`
- SVM의 `C`
- SVM의 `gamma`

그리고 각 조합에 대해 **학습 데이터 기준 교차검증(CV) 정확도** 를 계산해 행렬 및 히트맵으로 시각화했습니다.

#### 최적 조합

- `n_components = 50`
- `C = 10`
- `gamma = 0.001`
- CV Accuracy = 약 87.20%
- Test Accuracy = 약 88.48%

#### 이 결과의 의미

1. **차원 수가 너무 작으면 정보가 부족하다**
   `5`, `10`, `20`차원에서는 성능이 낮았습니다. 이는 분류에 필요한 정보가 충분히 남지 않았다는 뜻입니다.

2. **차원 수가 적당히 커질수록 성능이 좋아진다**
   `30~50`차원 구간에서 성능이 크게 개선됐습니다.

3. **너무 큰 차원 수는 다시 성능이 떨어질 수 있다**
   `70`, `100`, `150`차원에서 성능이 다시 약간 하락했습니다. 이는 추가 차원이 유용한 정보보다 노이즈를 더 많이 가져왔을 가능성을 시사합니다.

4. **현재 데이터의 최적 차원 수는 50 전후로 보인다**
   즉, '많이 남긴다고 무조건 좋은 것'이 아니라 적절한 정보량과 노이즈 제거 사이의 균형점이 존재합니다.

5. **`gamma=0.1`은 과도하게 크다**
   히트맵에서 해당 열 성능이 전반적으로 낮은 것은, 경계가 지나치게 복잡해져 일반화에 실패했기 때문으로 볼 수 있습니다.

### 11. 왜 CV Accuracy와 Test Accuracy를 함께 보는가?

- **CV Accuracy**: 학습 데이터 안에서 교차검증으로 측정한 성능
- **Test Accuracy**: 최종적으로 한 번만 확인하는 진짜 외부 평가 점수

CV Accuracy는 모델 선택에 사용하고, Test Accuracy는 최종 확인용으로 써야 데이터 누수를 줄일 수 있습니다.

이번 노트북에서는 이 원칙에 맞게, 먼저 CV로 최적 조합을 찾고 그 다음 테스트 데이터로 최종 성능을 확인했습니다.

### 15. Patch Transformer 기반 후속 실험

SVD/PCA + SVM 방식은 FordA 시계열을 저차원 벡터로 압축한 뒤 분류하는 방식입니다. 이 접근은 전통적인 머신러닝 기반의 강력한 기준선이지만, 시계열 내부의 국소적 파형 패턴과 멀리 떨어진 시점 간의 관계를 직접 학습하지는 못한다는 한계가 있습니다.

이를 보완하기 위해 후속 실험에서는 500개 타임스텝을 일정 길이의 패치로 나눈 뒤, 각 패치를 임베딩하여 Transformer Encoder에 입력하는 Patch Transformer 구조를 적용했습니다.

구조는 다음과 같습니다.

```text
Input Time Series
-> Patch Split
-> Conv1d Patch Embedding
-> CLS Token + Positional Embedding
-> Transformer Encoder
-> Binary Classifier
```

이번 실험에서는 다음 설정을 사용했습니다.

- 입력 길이: 500 타임스텝
- 패치 길이: 50
- 스트라이드: 25
- 임베딩 차원: 64
- Attention head 수: 4
- Transformer layer 수: 2
- 학습 epoch: 20

패치 길이를 50, 스트라이드를 25로 설정했기 때문에 인접 패치가 서로 겹치며 생성됩니다. 이는 패치 경계에서 정보가 끊기는 문제를 줄이고, 1D-CNN처럼 연속적인 국소 파형 특징을 더 안정적으로 반영하기 위한 설정입니다.

#### 실험 결과

- Best Validation Accuracy: 약 92.37%
- Patch Transformer Test Accuracy: 약 93.56%
- 기존 최적 SVD/PCA + RBF SVM Test Accuracy: 약 88.48%

혼동행렬 기준으로는 다음과 같은 결과를 보였습니다.

- 실제 비정상 681개 중 632개를 비정상으로 맞춤
- 실제 비정상 681개 중 49개를 정상으로 오분류
- 실제 정상 639개 중 603개를 정상으로 맞춤
- 실제 정상 639개 중 36개를 비정상으로 오분류

이는 기존 SVD/PCA + SVM 방식보다 테스트 정확도가 약 5%p 향상된 결과입니다.

#### 해석

Patch Transformer의 성능 향상은 FordA 데이터에서 단순한 전역 벡터 표현보다, 시간축을 따라 나타나는 국소적 패턴과 패치 간 관계를 함께 학습하는 방식이 더 적합할 수 있음을 보여줍니다.

특히 Conv1d patch embedding은 짧은 구간의 파형 형태를 특징으로 추출하고, Multi-Head Attention은 서로 떨어진 패치 사이의 연관성을 학습합니다. 따라서 특정 시점의 미세한 진동 패턴과 다른 구간의 이상 신호가 함께 분류에 영향을 주는 경우, Transformer 기반 구조가 더 유리할 수 있습니다.

다만 딥러닝 모델은 파라미터 수, 학습 epoch, patch length, stride, embedding dimension, attention head 수에 따라 성능 변동이 있을 수 있습니다. 따라서 현재 결과는 단일 설정에서의 강한 가능성을 보여주는 결과이며, 추가적인 하이퍼파라미터 튜닝과 반복 실험을 통해 안정성을 확인할 필요가 있습니다.

#### 결론

기존 SVD/PCA + SVM 실험은 FordA 분류 문제에서 약 88.48%의 기준 성능을 제공했습니다. 여기에 Patch Transformer를 적용한 결과 약 93.56%의 테스트 정확도를 얻었으며, 이는 시계열 데이터의 국소 패턴과 장기 의존성을 직접 학습하는 구조가 FordA 데이터에 더 적합할 수 있음을 시사합니다. 따라서 후속 연구에서는 1D-CNN/FCN 및 Patch Transformer 계열 모델을 함께 비교하여, 국소 특징 추출과 전역 문맥 학습이 분류 성능에 미치는 영향을 더 체계적으로 분석하는 것이 타당합니다.
